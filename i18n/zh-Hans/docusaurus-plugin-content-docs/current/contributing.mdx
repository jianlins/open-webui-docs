### 🌟 代码贡献指南

我们欢迎拉取请求。在提交之前，请：

1. 在[这里](https://github.com/open-webui/open-webui/discussions/new/choose)开启一个关于您想法的讨论。
2. 遵循项目的编码标准，并为新功能包含测试。
3. 根据需要更新文档。
4. 编写清晰、描述性的提交消息。

### 🛠 代码 PR 最佳实践：

1. **原子性 PR：** 确保您的 PR 小而集中，处理单一目标或任务。这有助于更容易的代码审查，并限制引入不相关问题的可能性。如果更改范围变得太大，考虑将它们分解为更小的、逻辑上独立的 PR。
2. **遵循现有代码约定：** 确保您的代码与项目现有的编码标准和实践保持一致。
3. **避免额外的外部依赖：** 未经事先讨论，不要包含额外的外部依赖。
4. **框架无关方法：** 我们的目标是保持框架无关。尽可能自己实现功能，而不是依赖外部框架或库。如果您对这种方法有疑问或建议，请随时讨论。

### 🌐 翻译和国际化

帮助我们使 Open WebUI 对更广泛的受众可用。在本节中，我们将指导您完成向项目添加新翻译的过程。

我们使用 JSON 文件存储翻译。您可以在 `src/lib/i18n/locales` 目录中找到现有的翻译文件。每个目录对应一种特定语言，例如，`en-US` 表示英语（美国），`fr-FR` 表示法语（法国）等。您可以参考 [ISO 639 语言代码](http://www.lingoes.net/en/translator/langcode.htm) 找到特定语言的适当代码。

要添加新语言：

- 在 `src/lib/i18n/locales` 路径中创建一个新目录，以适当的语言代码作为其名称。例如，如果您要添加西班牙语（西班牙）翻译，创建一个名为 `es-ES` 的新目录。
- 将美国英语翻译文件（从 `src/lib/i18n/locale` 中的 `en-US` 目录）复制到这个新目录，并根据您的语言更新 JSON 格式的字符串值。确保保留 JSON 对象的结构。
- 将语言代码及其相应标题添加到 `src/lib/i18n/locales/languages.json` 的语言文件中。

### 🚨 报告问题

注意到有问题？有想法？查看我们的 [Issues 标签](https://github.com/open-webui/open-webui/issues) 看看是否已经被报告或建议。如果没有，请随时开启一个新问题。报告问题时，请遵循我们的问题模板。这些模板旨在确保从一开始就提供所有必要的详细信息，使我们能够更有效地解决您的问题。

:::important

- **模板合规性：** 请注意，如果不遵循提供的问题模板，或根本不提供所请求的信息，很可能会导致您的问题被关闭而不予进一步考虑。这种方法对于维护问题跟踪的可管理性和完整性至关重要。

- **细节是关键：** 为确保您的问题被理解并能有效解决，提供全面的详细信息至关重要。描述应清晰，包括重现步骤、预期结果和实际结果。缺乏足够的细节可能会阻碍我们解决您问题的能力。

:::

### 🧭 支持范围

我们注意到与 Open WebUI 无直接关系但与其运行环境相关的问题有所增加，特别是 Docker 设置。虽然我们努力支持 Docker 部署，但了解 Docker 基础知识对于顺利体验至关重要。

- **Docker 部署支持**：Open WebUI 支持 Docker 部署。假定您熟悉 Docker。有关 Docker 基础知识，请参阅 [官方 Docker 文档](https://docs.docker.com/get-started/overview/)。

- **高级配置**：设置 HTTPS 的反向代理和管理 Docker 部署需要基础知识。有许多在线资源可用于学习这些技能。确保您拥有这些知识将大大增强您使用 Open WebUI 和类似项目的体验。
