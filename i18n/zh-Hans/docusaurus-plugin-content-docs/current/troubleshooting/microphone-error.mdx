确保您的应用程序具有适当的麦克风访问权限对于依赖音频输入的功能至关重要。本指南涵盖了如何管理和排除麦克风权限问题，特别是在安全上下文中。

## 理解安全上下文 🔒

出于安全原因，麦克风访问仅限于通过 HTTPS 或从 `localhost` 本地提供的页面。这一要求旨在通过确保数据通过安全通道传输来保护您的数据。

## 常见权限问题 🚫

Chrome、Brave、Microsoft Edge、Opera 和 Vivaldi 等浏览器，以及 Firefox，都限制在非 HTTPS URL 上的麦克风访问。当从同一网络内的另一设备访问站点时（例如，使用手机访问桌面服务器），这通常会成为一个问题。以下是您可以管理这些问题的方法：

### 非 HTTPS 连接的解决方案

1. **设置 HTTPS：**
   - 强烈建议配置您的服务器以支持 HTTPS。这不仅解决了权限问题，还增强了数据传输的安全性。

2. **临时浏览器标志（谨慎使用）：**
   - 这些设置强制您的浏览器将某些不安全的 URL 视为安全。这对于开发目的很有用，但会带来重大安全风险。以下是如何为主要浏览器调整这些设置：

   #### 基于 Chromium 的浏览器（例如，Chrome、Brave）
   - 打开 `chrome://flags/#unsafely-treat-insecure-origin-as-secure`。
   - 输入您的非 HTTPS 地址（例如，`http://192.168.1.35:3000`）。
   - 重启浏览器以应用更改。

   #### 基于 Firefox 的浏览器
   - 打开 `about:config`。
   - 搜索并修改（或创建）字符串值 `dom.securecontext.allowlist`。
   - 添加您的 IP 地址，用逗号分隔（例如，`http://127.0.0.1:8080`）。

### 考虑因素和风险 🚨

虽然浏览器标志提供了快速修复，但它们绕过了重要的安全检查，这可能会使您的设备和数据暴露于漏洞。始终优先考虑适当的安全措施，特别是在规划生产环境时。

通过遵循这些最佳实践，您可以确保您的应用程序正确访问麦克风，同时维护数据的安全性和完整性。
